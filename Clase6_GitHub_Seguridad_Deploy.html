Introducción · GitHub

¿Qué es GitHub?

Un servicio en la nube donde guardas, versionas y colaboras en código. Piénsalo como Google Drive, pero para proyectos de software — con superpoderes.

📦 Repositorio

Tu carpeta del proyecto

Cada proyecto vive en un repo. Contiene todos tus archivos y el historial completo de cambios.

🕐 Control de versiones

Máquina del tiempo

Cada commit es una foto del estado de tu código. Puedes volver atrás en cualquier momento.

🤝 Colaboración

Trabajo en equipo

Varias personas pueden trabajar en el mismo proyecto sin pisarse. Las ramas separan el trabajo de cada uno.

Introducción · Interfaz

La interfaz de GitHub

Cinco zonas clave que vas a usar en cada proyecto.

📁

Code

Explora archivos y carpetas del repo. Desde aquí editas directamente en el navegador.

🐛

Issues

Reporta bugs, propone mejoras o gestiona tareas pendientes del proyecto.

🔀

Pull Requests

Propone cambios para revisión antes de fusionarlos con el código principal.

⚙️

Actions

Automatiza tareas: tests, deploys y más se ejecutan solos cuando subes código.

🔧

Settings

Configura el repo: visibilidad, colaboradores, GitHub Pages y secretos de entorno.

💡 Flujo básico: escribes código → git add → git commit → git push → tu código aparece en el repo online.

Introducción · Comandos

Diccionario Git

Los comandos que vas a usar en cada sesión de trabajo.

git init
Inicializa un repositorio Git en tu carpeta local.
Solo se hace una vez por proyecto.
git add .
Marca todos los archivos modificados como "listos para guardar".
El punto (.) significa "todo". Puedes poner un archivo específico.
git commit -m "..."
Saca una foto del estado actual del código y le pone nombre.
El mensaje describe qué cambiaste. Ej: "agrego formulario de contacto".
git push
Sube los commits locales al repositorio en GitHub.
Tu código pasa de tu máquina a la nube.
git pull
Descarga los cambios que hay en GitHub hacia tu máquina.
Úsalo siempre antes de empezar a trabajar en equipo.
git status
Muestra qué archivos fueron modificados y cuáles están listos para commit.
Tu comando "¿en qué estado estoy?" antes de hacer cualquier cosa.
git clone <url>
Descarga una copia completa de un repositorio remoto a tu máquina.
La URL la sacas del botón verde "Code" en GitHub.

Bloque 01 · Seguridad

El error #1: API Keys públicas

Subir una clave a un repo público es como dejar las llaves de tu casa pegadas en la puerta. Los bots la encuentran en segundos.

!Se quedan en el historial. Borrar el archivo no basta: la clave sigue en commits anteriores.
!Se escanean automáticamente. Existen bots que rastrean GitHub buscando claves expuestas 24/7.
!El costo es real. Una key de OpenAI o AWS filtrada puede generar cobros enormes en horas.
~/mi-proyecto
# ❌ Nunca hagas esto const apiKey = "sk-proj-9fA2k...REAL"; $ git add . && git commit -m "app" $ git push origin main ⚠ key detectada por un bot en 11s

Bloque 01 · Buenas prácticas

Cómo proteger tus secretos

❌ Lo que NO se sube

  • API Keys o tokens directamente en el código
  • Archivos .env con credenciales
  • Contraseñas de bases de datos o conexiones
  • Claves privadas (.pem, .key)

✓ La forma correcta

  • Guardar las claves en un archivo .env
  • Agregar .env al .gitignore
  • Usar variables de entorno o GitHub Secrets
  • Si se filtró: rotar la clave de inmediato
.gitignore
# Ignorar secretos antes del primer push .env .env.local # nunca llegará a GitHub

Bloque 02 · Antigravity

¿Qué es Antigravity?

Un entorno donde la IA no solo escribe código, sino que actúa sobre tu repositorio: crea, organiza y versiona de forma autónoma — sin que tú copies y pegues archivo por archivo.

→Acceso interno al repo. Trabaja directamente sobre GitHub, no como una caja aislada.
→Creación autónoma. Genera la estructura de archivos y carpetas por sí mismo.
→Menos fricción. Pasas de la idea al repositorio sin pasos manuales repetitivos.

Bloque 03 · Deploy

GitHub Pages: tu HTML en internet

Hosting gratuito y directo desde tu repositorio. Subes un index.html y GitHub te entrega una URL pública.

✓Gratis y sin servidor. No necesitas contratar hosting ni configurar nada complejo.
✓Un solo archivo basta. Con index.html en la raíz ya tienes una web.
✓URL automática. Tu sitio vive en usuario.github.io/repo.

Bloque 03 · Paso a paso

Publicar en 4 pasos

1

Crea el repositorio

En GitHub, nuevo repo público. Sube tu archivo index.html a la raíz.

2

Entra a Settings → Pages

Dentro del repo, abre Settings y busca la sección Pages en el menú lateral.

3

Selecciona la rama

En Source elige la rama main y la carpeta / (root). Guarda.

4

Abre tu URL

En 1–2 minutos GitHub publica el sitio en usuario.github.io/repo. ¡Listo!
